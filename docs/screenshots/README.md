# Screenshots

This directory contains screenshots for the README documentation.

## Required Screenshots

Please capture the following screenshots and save them with the specified filenames:

### 1. `directory.png`
**Public Meeting Directory**
- Navigate to the main page (home/browse view)
- Show the map with meeting markers visible
- Include the filter panel on the left
- Capture at desktop resolution (1280x800 or similar)

### 2. `admin-dashboard.png`
**Admin Dashboard**
- Log in as admin
- Navigate to the Dashboard/Home tab
- Show the statistics cards (Total Meetings, States, etc.)
- Include the scraping controls if visible
- Capture the activity log or recent meetings section

### 3. `coverage.png`
**Coverage Analysis**
- Navigate to the Coverage tab in admin
- Show the state coverage table
- Include the summary statistics at the top
- Show both covered and uncovered states

### 4. `dev-docs.png`
**Developer Documentation**
- Open the Documentation modal (Docs button)
- Navigate to the iOS or Android guide tab
- Show code examples visible
- Include the sidebar navigation

## Screenshot Guidelines

- **Resolution**: 1280x800 or higher recommended
- **Format**: PNG preferred for clarity
- **Theme**: Use dark mode for consistency (or light mode if that's the default)
- **Sensitive Data**: Blur or remove any personal information
- **Browser**: Hide browser chrome if possible, or use a clean browser profile

## Generating Screenshots

You can use browser developer tools to capture full-page screenshots:
1. Open DevTools (F12)
2. Press Ctrl+Shift+P (Cmd+Shift+P on Mac)
3. Type "screenshot" and select "Capture full size screenshot"

Or use the built-in screenshot tools in your operating system.
